1. Modular Architecture

Separate the project into independent modules.

Instead of one large application, create separate services for:

* Frontend (React)
* Backend API
* Market Data Collector
* Scheduler / Cron Jobs
* Database Layer
* Authentication
* Backup Service
* Notification Service
* Logging Service
* Deployment Service

Every service should have a clear responsibility.

2. Symbol Master Database

* Exchange
* Symbol
* Company Name
* ISIN
* Instrument Type
* Sector
* Industry
* Market Cap
* Lot Size
* Tick Size
* Freeze Quantity
* F&O Eligible
* Index Membership
* Corporate Action History
* Listing Date
* Delisting Status

Every table should reference this master table.

3. Corporate Actions

Design automatic handling of:

* Stock Splits
* Bonus Issues
* Rights Issues
* Mergers

4. Database Versioning

Design database migration support.

5. Scanner Versioning
