How ATM Must Be Determined

ATM must always be calculated separately for every underlying stock or index.

For every symbol:

Example:

RELIANCE Futures

The Future Close Price determines where ATM is located.

Example:

Future Close = 2517

Available CE strikes:

2500

2520

2540

2560

Nearest strike to the Future Close is

2520

Therefore

ATM = 2520

This calculation must happen independently for every stock.

Do NOT use Spot Price.

Use the Futures Close Price available in the Options CSV.

Scanner Behaviour

When a scan is executed

The application must

Loop through every symbol in the selected Universe.

For every symbol

Calculate ATM dynamically.

Convert ATM into the requested ITM or OTM strike according to CE or PE rules.

Find the matching option contract.

Evaluate every scanner condition using that contract.

This must work for every symbol independently.

Different stocks have different Futures prices.

Therefore every stock has a different ATM strike.

ATM is never a fixed value.

⸻

Important Rules

Do NOT assume fixed strike intervals.

Some symbols use

₹10 intervals

Some use

₹20

₹25

₹50

₹100

Always use the actual strike prices available in the CSV.

Never calculate strike prices using mathematical intervals.

Always move to the previous or next available strike in the sorted strike list.

Edge Cases

User selects ITM +2

Do not produce incorrect results.

If expiry has no matching option chain

Skip the symbol.

If Futures data is missing

Skip the symbol.

Do not crash.

Expected Result

When the user selects:

Universe = Nifty50

Side = CE

Expiry = 2026-06-30

Strike = ITM +1

The application should:

1. Process every Nifty50 symbol.
2. Find that symbol’s Futures Close Price.
3. Determine the nearest available ATM strike.
4. Move one available strike below ATM.
5. Select that CE option contract.
6. Evaluate the scanner using that contract.

Similarly, when the user selects:

Universe = Futures

Side = PE

Strike = OTM -2

The application should:

1. Process every Futures symbol.
2. Determine the ATM strike from the Futures Close Price.
3. Move two available strikes below ATM.
4. Select the corresponding PE option contract.
5. Run the scanner using that option.

The implementation must be completely dynamic, use only the CSV data, support all symbols and expiries, and produce accurate results without any hardcoded strike values or assumptions.
