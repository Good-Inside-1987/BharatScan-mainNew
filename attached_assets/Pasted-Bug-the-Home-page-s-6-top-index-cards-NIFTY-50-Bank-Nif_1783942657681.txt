Bug: the Home page's 6 top index cards (NIFTY 50, Bank Nifty, Fin Nifty,
Midcap Nifty, Nifty Next 50, Sensex) always show "Not in CSV" even with
Fyers connected and the market live. Root cause: IndexCard in
artifacts/bharatscan/src/pages/Home.tsx only ever resolves its price from
`histories` (CSV-upload or the Scanner's ohlcv_daily DB-bridge) via
lookupIndex(), or from `optionsData` via lookupFuturesIndex() — both are
end-of-day/historical sources. Neither the automated symbol-master sync
nor the EOD/intraday pipeline ever includes the raw index instruments
themselves (only individual equity/F&O stocks get synced), so these 6
cards have no automated data source at all — a leftover from the app's
original pre-automation, CSV-only design (hence the literal "Not in CSV"
label).

The backend already has real-time prices for 5 of these 6 indices
flowing in memory: liveOptionsTracker.ts subscribes each configured
index's underlying Fyers symbol (e.g. NSE:NIFTY50-INDEX) as part of ATM
option tracking, cached via liveFeedService's getLiveQuote(). The generic
GET /api/market-data/quotes route (backed by getLiveQuotes()) already
exposes this, and the frontend even has an unused helper for it —
apiGetMarketQuotes() in lib/api.ts — that was never wired to anything.
This fix is almost entirely additive: no new backend endpoint needed.

2. Import `apiGetMarketQuotes` and its `ApiLiveQuote` type from

3. In the main Home() component, alongside the existing `useData()` /
   `useHomeIndexSource()` / `useIndexSlots()` calls:

     const [liveIndexQuotes, setLiveIndexQuotes] = useState<Record<string, ApiLiveQuote>>({});

     useEffect(() => {
       const fyersSymbols = ALL_INDEX_DEFS
         .filter((d) => d.fyersIndexSymbol)
         .map((d) => d.fyersIndexSymbol!);
       if (!fyersSymbols.length) return;

       let cancelled = false;
       const poll = async () => {
         try {
           const { quotes } = await apiGetMarketQuotes(fyersSymbols);
           if (cancelled) return;
           const bySymbol = new Map(quotes.map((q) => [q.symbol, q]));
           const byKey: Record<string, ApiLiveQuote> = {};
           for (const d of ALL_INDEX_DEFS) {
             if (d.fyersIndexSymbol && bySymbol.has(d.fyersIndexSymbol)) {
               byKey[d.key] = bySymbol.get(d.fyersIndexSymbol)!;
             }
           }
           setLiveIndexQuotes(byKey);
         } catch {
           // No broker connected, market closed, or a transient error —
           // leave existing state as-is so IndexCard falls back to
           // Futures/Spot below rather than showing a jarring blank.
         }
       };

       poll();
       const id = setInterval(poll, 10_000);
       return () => { cancelled = true; clearInterval(id); };
     }, []);

   Fetch all 5 known index symbols regardless of which 6 are in the
   user's active slots — slot-switching is instant then, with no refetch
   needed, and the constant list is tiny.

5. In IndexCard's props type, add `liveQuote?: ApiLiveQuote`. In its

     const { data, sourceLabel } = useMemo(() => {
       if (liveQuote) {
         const pct = liveQuote.close > 0 ? ((liveQuote.ltp - liveQuote.close) / liveQuote.close) * 100 : 0;
         const pts = liveQuote.close > 0 ? liveQuote.ltp - liveQuote.close : 0;
         return { data: { value: liveQuote.ltp, pct, pts, found: true }, sourceLabel: "Live" };
       }
       if (def.futuresSymbol && homeIndexSource === "futures") {
         // ...unchanged...
       }
       // ...rest unchanged...
     }, [liveQuote, histories, def.aliases, def.futuresSymbol, latestDate, optionsData, asOfOptionsDate, homeIndexSource]);

   liveQuote.close is Fyers' actual previous-day close (verified in
   server/src/adapters/fyers.ts — mapped from prev_close_price), so this

above them. When the broker's disconnected or a symbol has no live quote
yet, it falls through to exactly the same Futures/Spot/"Not in CSV"
behavior as today, unchanged.
