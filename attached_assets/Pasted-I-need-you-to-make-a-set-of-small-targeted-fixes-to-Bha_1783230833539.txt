FIX 1 — Global Express error handler

In server/src/index.ts, add this as the VERY LAST middleware,
after all routes and after app.listen(...). This is critical for
debugging Angel API failures — without it, any unhandled error
crashes the request silently:

tsapp.use((err: Error, _req: express.Request, res: express.Response, _next: express.NextFunction) => {
  console.error('[SERVER ERROR]', new Date().toISOString(), err.message);
  if (process.env.NODE_ENV !== 'production') {
    console.error(err.stack);
  }
  if (res.headersSent) return;
  res.status(500).json({ error: err.message || 'Internal server error' });
});

Also add import express from 'express' types if not already imported
so the express.Request / express.Response types resolve correctly.

FIX 3 — SPA catch-all route (fixes 404 on page refresh)

In server/src/index.ts, after all /api/* routes are registered
but BEFORE the global error handler from Fix 1, add:

tsimport path from 'path';
import { fileURLToPath } from 'url';

const __dirname = path.dirname(fileURLToPath(import.meta.url));

// Serve built frontend in production
if (process.env.NODE_ENV === 'production') {
  const distPath = path.resolve(__dirname, '../../artifacts/bharatscan/dist/public');
  app.use(express.static(distPath));
  app.get('*', (_req, res) => {
    res.sendFile(path.join(distPath, 'index.html'));
  });
}

This fixes the problem where refreshing the browser at /portfolio,
/paper-trading, or any non-root page returns a 404 on Oracle.
In Replit dev mode, Vite handles routing so this block is skipped.
