This is genuinely complete now. Only a handful of things remain:
Will Actually Break Something
pm2 startup and pm2 save commands are missing from Phase 3. PM2 keeping the app running 24/7 is mentioned throughout the document, but without running these two specific commands on Oracle, PM2 stops everything on every server reboot. Oracle VMs do reboot occasionally for maintenance. This is a one-time setup step that must be in Phase 3.
GitHub Actions needs more than just the SSH private key in GitHub Secrets. It also needs the Oracle server's IP address and the username stored as secrets (ORACLE_HOST, ORACLE_USER). The deploy.yml file references these to know where to connect. The document only mentions the SSH key — someone following it will create the key pair but have no host or user configured.
Oracle's default username on ARM instances is opc, not ubuntu. Most Linux tutorials use ubuntu so this is a common confusion point. The SSH key setup and GitHub Actions config both need to use opc as the user, otherwise the SSH connection fails.
Will Cause a Subtle Problem
Market data tables (60GB growing) and user data (scans, portfolio, alerts) are planned to go into the same bharatscan.db file. If the market data layer corrupts the database during a large write or failed sync, user data is lost too. These should be two separate SQLite files from the start — market.db for OHLCV/options, app.db for user data. The backup strategy would also be cleaner this way.
Minor
The GitHub Actions workflow trigger should specify the main branch only. As written, "triggers on every GitHub push" means pushing any feature branch to GitHub would deploy to production Oracle. Should be on: push: branches: [main].
The holiday list has a bootstrap problem. The scheduler skips holidays using the stored holiday list, but on Day 1 the list is empty. The first-run script needs to fetch and store holidays before the scheduler starts, not after.
Everything else is correct and complete. This document is ready to use as-is for the implementation. All previous rounds of feedback are properly incorporated.
