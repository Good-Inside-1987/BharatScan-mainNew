# Prompt 23 — Stop marking failed EOD/intraday syncs as "completed"

## Bug
In `server/src/services/syncJobs.ts`, both `runEodSyncJob` and
`runIntradaySyncJob` call:

```ts
finishSyncLog(logId, "completed", stats);
```

unconditionally — even when `stats.failed` equals the full symbol count and
`stats.completed` is 0.

`getLastSuccessDate(jobName)` only looks at rows where `status = 'completed'`.
So a run that failed for every symbol still gets treated as a successful
sync for that date. `catchUpScheduler.ts` then sees "already caught up" and
never retries — this is why the app can report "No missed trading days
found on startup" while `ohlcv_daily` has almost no real data.

## Fix
In `runEodSyncJob` (around the "Retention cleanup" comment near the end of
the function), replace:

```ts
    finishSyncLog(logId, "completed", stats);
    console.log(
      "[syncJobs] EOD sync done — completed=%d skippedBudget=%d failed=%d",
      stats.completed, stats.skippedBudget, stats.failed
    );
    return stats;
```

with:

```ts
    const runStatus: "completed" | "failed" = stats.failed === 0 ? "completed" : "failed";
    finishSyncLog(
      logId,
      runStatus,
      stats,
      runStatus === "failed" ? `${stats.failed} of ${symbols.length} symbols failed` : undefined
    );
    console.log(
      "[syncJobs] EOD sync done — status=%s completed=%d skippedBudget=%d failed=%d",
      runStatus, stats.completed, stats.skippedBudget, stats.failed
    );
    return stats;
```

Do the exact same replacement in `runIntradaySyncJob`'s equivalent block
(same comment, same `finishSyncLog(logId, "completed", stats);` line, same
console.log pattern with `"[syncJobs] Intraday sync done — ..."`).

## Do NOT touch
`optionsDataService.ts`'s `runOptionsSyncJob` already has an equivalent
"failed" branch (checks `completed === 0`) — leave it as is.

## Verify after applying
1. `grep -n "finishSyncLog(logId, \"completed\", stats)" server/src/services/syncJobs.ts`
   should return **zero** matches (both call sites now use the `runStatus` variable).
2. Trigger a manual EOD sync run on a day where the broker session is NOT
   connected — confirm `sync_log` shows `status='failed'` for that
   `target_date`, and that a subsequent startup catch-up run for that date
   is no longer skipped.
