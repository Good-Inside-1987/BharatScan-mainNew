FIX 1 — Correct the option-chain expiry parameter name in fyers.ts.
Context: getOptionChain() is currently sending the requested expiry date under a timestamp query parameter. Fyers' option-chain API actually expects the date under a parameter literally named expiry (format YYYY-MM-DD); timestamp is a separate, unrelated parameter that should be left unset. Because of this mismatch, every call that passes a specific expiry (including all cached-expiry lookups from liveOptionsTracker.ts) fails with "Please provide valid expiry" — this is why NIFTY/BANKNIFTY/FINNIFTY/SENSEX/MIDCPNIFTY option chains never load once the expiry is cached.
In server/src/adapters/fyers.ts, inside getOptionChain(), find:
jsif (expiry) url.searchParams.set("timestamp", expiry);
Replace it with:
jsif (expiry) url.searchParams.set("expiry", expiry);
Do not change anything else in this method — the symbol param, the response parsing, or the expiries extraction logic are all correct as-is.
FIX 2 — Correct the live feed subscribe/unsubscribe message format and connection URL in liveFeedService.ts.
Context: The current subscribe/unsubscribe messages use a { T: "SUB_L1", L1: 1, SLIST: [...] } shape, which does not match Fyers' documented data-socket protocol at all. Fyers' v3 data socket expects { "symbol": [...], "type": "symbolUpdate" } (or "type": "lite" for LTP-only updates) as the subscribe message. Separately, the connection URL currently includes a type=symbolData query parameter that isn't part of Fyers' documented connection handshake (per their docs, only the access token authorizes the connection — the type field belongs in the post-connect subscribe message, not the URL). This may be contributing to the Unexpected server response: 500 handshake failures.
In server/src/services/liveFeedService.ts, find:
jsfunction buildSubscribeMessage(symbols: string[]): string {
  return JSON.stringify({ T: "SUB_L1", L1: 1, SLIST: symbols });
}

function buildUnsubscribeMessage(symbols: string[]): string {
  return JSON.stringify({ T: "UNSUB_L1", L1: 1, SLIST: symbols });
}
Replace with:
jsfunction buildSubscribeMessage(symbols: string[]): string {
  return JSON.stringify({ symbol: symbols, type: "symbolUpdate" });
}

function buildUnsubscribeMessage(symbols: string[]): string {
  return JSON.stringify({ symbol: symbols, type: "symbolUpdate", unsubscribe: true });
}
(Note: if you find Fyers' docs specify a different unsubscribe shape once you look at the live response, flag it back to me rather than guessing — the subscribe shape above is confirmed, the unsubscribe shape is my best inference from the same protocol and should be verified against the first real response.)
Then find:

Replace with:
jsconst url = `${FYERS_DATA_WS_URL}?access_token=${encodeURIComponent(authToken)}`;
