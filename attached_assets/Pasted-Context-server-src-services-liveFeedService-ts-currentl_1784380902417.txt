Context: server/src/services/liveFeedService.ts currently connects directly to
wss://api.fyers.in/socket/v2/data/ using the `ws` package with a JSON subscribe
protocol. This is wrong — Fyers' real v3 data feed uses:
  - endpoint wss://socket.fyers.in/hsm/v1-5/prod
  - a proprietary BINARY framing protocol (not JSON)
This can't be reliably reimplemented from scratch without live testing access.
Instead, we will bridge to it via the official Python SDK (`fyers-apiv3`, which
already implements this binary protocol correctly), run as a child process.

1. Add server/python/requirements.txt containing: fyers-apiv3

2. Create server/python/fyers_ws_bridge.py:
   - Reads FYERS_APP_ID and FYERS_ACCESS_TOKEN from environment variables
     (never hardcode credentials).
   - Builds access_token as "{FYERS_APP_ID}:{FYERS_ACCESS_TOKEN}".
   - Uses fyers_apiv3.FyersWebsocket.data_ws.FyersDataSocket with:
       - on_connect: print a single line `{"__control__":"connected"}` to
         stdout and flush immediately, so the Node parent can detect a
         successful handshake.
       - on_message: for every tick dict received, print it as one line of
         JSON to stdout (json.dumps(message) + "\n"), flushed immediately.
         Do NOT transform field names — pass the raw dict through so the
         Node side can inspect real field names during testing rather than
         guess at them.
       - on_error: print `{"__control__":"error","detail": <str(err)>}` to
         stdout, flushed.
       - on_close: print `{"__control__":"closed"}` to stdout, flushed, then
         exit the process.
   - Reads control commands as newline-delimited JSON from stdin in a
     background thread, without blocking the websocket:
       - {"action":"subscribe","symbols":[...],"mode":"full"} ->
         ws.subscribe(symbols, litemode=False) then ws.mode(ws.FullMode) as
         shown in the SDK's own sample/datasocket.js and Python quickstart.
       - {"action":"unsubscribe","symbols":[...]} -> ws.unsubscribe(symbols)
   - Calls ws.connect() last, after all callbacks are wired.
   - Wrap all Fyers SDK calls in try/except and print
     {"__control__":"error","detail":...} rather than letting the process
     crash silently — a dead sidecar must be visible to the Node parent.

3. Rewrite server/src/services/liveFeedService.ts's connection lifecycle
   ONLY (do not touch the SubscriptionManager class, quoteCache,
   candleState/recordTick/finalizeCandle/upsertOhlcvCandle/
   upsertOptionsCandle, or the public exported functions' signatures):
   - Replace `import WebSocket from "ws"` connection logic in connect()
     with child_process.spawn("python3", ["server/python/fyers_ws_bridge.py"],
     { env: { ...process.env, FYERS_APP_ID: session.appId,
       FYERS_ACCESS_TOKEN: session.accessToken } }).
   - Pipe child.stdout through Node's readline interface, one JSON line at a
     time. For each parsed line:
       - if it has "__control__":"connected" -> treat exactly like the
         current socket.on("open") handler (clear handshake timer, reset
         reconnectAttempt, re-send subscriptions).
       - if "__control__":"error" or "closed" -> log it and call the
         existing scheduleReconnect().
       - otherwise -> this is a raw tick dict. Log the FIRST ten distinct
         raw shapes received (console.log with a one-time-per-run guard) so
         we can see real Fyers field names before hardening
         parseTickMessage's field mapping. Do not silently drop fields we
         don't recognize yet.
   - Replace buildSubscribeMessage/buildUnsubscribeMessage's `sendIfOpen`
     usage with writing a JSON control line + "\n" to child.stdin instead
     of ws.send().
   - Replace isConnected()'s `ws.readyState` check with a boolean flag set
     true on the "connected" control line and false on "closed"/process exit.
   - Replace disconnect()'s ws.close() with child.kill() and clear the flag.
   - Keep HANDSHAKE_TIMEOUT_MS behavior: if no "connected" control line
     arrives within HANDSHAKE_TIMEOUT_MS of spawning, kill the child process
     and call scheduleReconnect(), exactly like the current timer does for
     the raw socket.
   - If the python3 binary or the fyers_apiv3 package is missing, spawn will
     fail — catch this, log a clear one-line error telling the operator to
     run `pip install -r server/python/requirements.txt`, and call
     scheduleReconnect() rather than crashing the server process.
