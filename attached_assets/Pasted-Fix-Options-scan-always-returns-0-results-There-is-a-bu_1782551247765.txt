Fix: Options scan always returns 0 results
There is a bug in artifacts/bharatscan/src/pages/Index.tsx that causes the options scanner to always return zero results, even when matching contracts exist. Stocks scan works fine. The fix is a single function addition plus 5 call-site replacements — do not rewrite anything else.

Root Cause
The series state variable defaults to "EQ". Every runScan() call passes series: resolveSeriesFilter(series) which resolves to ["EQ"]. Inside runScan() in screener.ts, this becomes a filter:


ts
const seriesFilter = new Set(["EQ"]);
for (const h of histories) {
  if (seriesFilter && !seriesFilter.has(h.series)) continue; // skips non-EQ
}
But buildOptionHistories() in options.ts creates synthetic SymbolHistory objects with series: "OPT", not "EQ":


ts
out.push({ symbol: `${symbol} ${side}`, series: "OPT", bars });
So when options mode is active, the ["EQ"] series filter silently rejects every single option history, producing 0 matches regardless of the scan conditions. Stocks are unaffected because their histories have series: "EQ" from the NSE bhavcopy CSV.

The Fix — do exactly this, nothing more
Step 1: In artifacts/bharatscan/src/pages/Index.tsx, find the existing resolveSeriesFilter function (around line 170). It looks like this:


ts
function resolveSeriesFilter(sel: string): string[] | undefined {
  if (sel === "ALL") return undefined;
  if (sel === "EQ") return ["EQ"];
  if (sel === "ETF") return seriesList.filter((s) => /ETF/i.test(s));
  if (sel === "BOND") {
    const bondCodes = new Set(["GB", "GS", "SG", "TB", ...]);
    return seriesList.filter((s) => bondCodes.has(s) || /BOND|DEBT|GSEC|GILT/i.test(s));
  }
  return [sel];
}
Immediately after the closing } of that function, add this new function:


ts
// In options mode, filteredHistories are already fully filtered by
// buildOptionHistories (symbol universe + expiry + side). The series field on
// those synthetic histories is "OPT", not "EQ". Passing resolveSeriesFilter(series)
// (which returns ["EQ"]) to runScan filters out every option history, giving 0
// results. Return undefined in options mode so runScan skips the series gate.
function activeSeriesFilter(): string[] | undefined {
  if (scanMode === "options") return undefined;
  return resolveSeriesFilter(series);
}
Step 2: In the same file, find every call to runScan() or runBacktest() that passes series: resolveSeriesFilter(series) and change it to series: activeSeriesFilter(). There are 4 such places:
* Inside runScanNow() — the main scan call
* Inside runBacktestNow() — the backtest call
* Inside runBacktestNow() — the auto-select latest day runScan call
* Inside selectBacktestDay() — the per-day recompute call
Change each one from:


ts
series: resolveSeriesFilter(series),
to:


ts
series: activeSeriesFilter(),
Step 3: Also in the same file, find the CSV export block (inside the backtest history download button's onClick). It has a local variable:


ts
const seriesFilter = resolveSeriesFilter(series);
Change it to:


ts
const seriesFilter = activeSeriesFilter();

Verification
After the change, search the file for resolveSeriesFilter(series) — it should appear zero times as a direct call site (it will still appear inside activeSeriesFilter's own body as return resolveSeriesFilter(series), which is correct). The word activeSeriesFilter should appear in 5 places: its definition plus the 5 call sites replaced above.
Do not modify screener.ts, options.ts, DataContext.tsx, or any other file. The bug is entirely in Index.tsx.
